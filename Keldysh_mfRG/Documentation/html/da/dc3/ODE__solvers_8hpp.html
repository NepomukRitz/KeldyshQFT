Explicit RK4 using non-constant step-width determined by substitution, allowing to save state at each Lambda step. Allows for checkpointing: If last parameter it_start is given, ODE solver starts at this iteration (to continue previously canceled computation). If it_start is not given (see overload below), full flow is computed.

Template Parameters
T: type of object to be integrated (usually State, will currently only work for state, since result is saved using functions from hdf5_routines.h, which only support State)
Parameters
y_fin: reference to object into which result is stored (final State)
x_fin: final value of the integration variable (final Lambda)
y_ini: initial value of the integrated object (initial State)
x_ini: initial value of the integration variable (initial Lambda)
rhs: right hand side of the flow equation to be solved
subst: substitution to generate non-equidistant flow grid
resubst: resubstitution
N_ODE: number of ODE steps (will be increased by adding points at interesting Lambda values)
filename: output file name
it_start: Lambda iteration at which to start solving the flow
